REST API REQUIRED
	HEADER
		potremmo fare un "parcer" da nome parziale di uno di questi tre oggetti a id e tenere le get by id, altrimenti
			-get book by title 		/books/byTitle/{title}
			-get event by name		/events/byName/{eventName}
			-get author by name		/author/byName/{name}
			-get user				/user
				serve per prendere le informazioni del proprio utente, ad esempio il nome da mostrare nell'header.
				si basa sui cookie, quindi non ha bisogno di un id

	INDEX_PAGE
		-get book by sponsored field		/books/sponsored
			aggiungere campo sponsored alle seed di book

		-get event by date					/events/byDate/{date}
			DATE FORMAT: "gg-mm-yyyy"

		-get book by publication_date		/books/byPublicationDate/{date}
			aggiungere campo pubblication_date alle seed di book
	BOOK PAGE
		-get book by genre			/books/byGenre/{genre}
		-get book by theme			/books/byTheme/{theme}
		-get book by sold copies	/books/bySoldCopies
			aggiungere campo sold copies alle seed di book
		-get book by author			/books/byAuthor/{author}
	EVENT PAGE
		-get event by category		/events/byCategory/{category}
			aggiungere category al seed di event
